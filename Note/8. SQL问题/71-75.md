## 71. 如何向表格添加列？

可以使用 `ALTER TABLE` 语句和 `ADD COLUMN` 子句向现有表中添加新的列。

**语法:**

```sql
ALTER TABLE table_name
ADD COLUMN column_name data_type column_constraints;
```

* **`ALTER TABLE`**:  关键字，表示要修改表结构。
* **`table_name`**: 要添加列的表的名称。
* **`ADD COLUMN`**:  关键字，表示要添加新的列。
* **`column_name`**:  新列的名称。
* **`data_type`**:  新列的数据类型，例如 `INT`, `VARCHAR`, `DATE` 等。
* **`column_constraints`**: 可选的，用于定义新列的约束，例如 `NOT NULL`, `UNIQUE`, `DEFAULT` 等。

**示例:**

```sql
-- 向 employees 表添加名为 email 的列，数据类型为 VARCHAR(255)
ALTER TABLE employees
ADD COLUMN email VARCHAR(255);
```


## 72. 如何重命名表的列？

可以使用 `ALTER TABLE` 语句和 `RENAME COLUMN` 子句来重命名现有表的列。

**语法:**

```sql
ALTER TABLE table_name
RENAME COLUMN old_column_name TO new_column_name;
```

* **`ALTER TABLE`**:  关键字，表示要修改表结构。
* **`table_name`**:  要重命名列的表的名称。
* **`RENAME COLUMN`**:  关键字，表示要重命名列。
* **`old_column_name`**:  要重命名的列的当前名称。
* **`new_column_name`**:  列的新名称。

**示例:**

```sql
-- 将 employees 表中的 name 列重命名为 full_name
ALTER TABLE employees
RENAME COLUMN name TO full_name;
```


## 73. 如何从表中删除列？

可以使用 `ALTER TABLE` 语句和 `DROP COLUMN` 子句从现有表中删除列。

**语法:**

```sql
ALTER TABLE table_name
DROP COLUMN column_name;
```

* **`ALTER TABLE`**:  关键字，表示要修改表结构。
* **`table_name`**:  要删除列的表的名称。
* **`DROP COLUMN`**:  关键字，表示要删除列。
* **`column_name`**:  要删除的列的名称。

**示例:**

```sql
-- 从 employees 表中删除 email 列
ALTER TABLE employees
DROP COLUMN email;
```

**注意:** 

* 删除列是一个不可逆的操作，请谨慎操作。 
* 一些数据库系统可能不支持 `DROP COLUMN` 语句，或者需要额外的语法或权限。


## 74. 如何选择表中的所有偶数或奇数的记录？

可以使用模运算符 (`%` 或 `MOD`) 来判断一个数是偶数还是奇数。

* **偶数:**  一个数除以 2 的余数为 0。
* **奇数:**  一个数除以 2 的余数为 1。

**示例:**

假设要查询 `products` 表中所有 `product_id` 为偶数的记录:

```sql
-- 使用 % 运算符 (适用于大多数数据库系统)
SELECT * FROM products WHERE product_id % 2 = 0;

-- 使用 MOD 函数 (适用于 MySQL, PostgreSQL)
SELECT * FROM products WHERE MOD(product_id, 2) = 0;
```

要查询 `product_id` 为奇数的记录，只需将条件改为 `product_id % 2 = 1` 或 `MOD(product_id, 2) = 1` 即可。


## 75. 查询时如何防止重复记录？

可以使用以下两种方法来防止查询结果中出现重复记录：

* **使用 `DISTINCT` 关键字:**  `DISTINCT` 关键字可以用于 `SELECT` 语句中，用于返回唯一的结果集，消除重复的行。

    ```sql
    -- 查询 employees 表中所有唯一的 department_id
    SELECT DISTINCT department_id FROM employees;
    ```

* **为表创建唯一约束:**  可以在创建表时或使用 `ALTER TABLE` 语句为表中的一个或多个列创建唯一约束。 唯一约束可以确保这些列中的值组合在表中是唯一的，从而防止插入重复记录。

    ```sql
    -- 创建 employees 表时为 email 列创建唯一约束
    CREATE TABLE employees (
        ...,
        email VARCHAR(255) UNIQUE,
        ...
    );

    -- 为现有表 employees 的 email 列添加唯一约束
    ALTER TABLE employees
    ADD CONSTRAINT unique_email UNIQUE (email);
    ```

**选择哪种方法取决于具体的需求:**

* 如果只是想在查询结果中消除重复记录，使用 `DISTINCT` 关键字更方便。
* 如果需要在数据层面上保证数据的唯一性，则应该使用唯一约束。
