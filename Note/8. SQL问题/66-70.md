## 66. DELETE 和 TRUNCATE 的区别是什么？

`DELETE` 和 `TRUNCATE` 都是用于删除表中数据的 SQL 语句，但它们在操作方式和功能上有所不同：

| 特性        | DELETE                                                                     | TRUNCATE                                                                       |
|--------------|---------------------------------------------------------------------------|---------------------------------------------------------------------------------|
| 功能        | 删除表中的指定行                                                             | 删除表中的所有行                                                                 |
| 条件        | 可以使用 `WHERE` 子句指定删除条件                                               | 不支持 `WHERE` 子句                                                               |
| 速度        | 相对较慢，因为它会逐行删除数据，并记录日志                                   | 相对较快，因为它会直接删除数据页，并且不记录日志                               |
| 事务        | 是一个 DML 语句，操作可以回滚                                                | 是一个 DDL 语句，操作不能回滚                                                 |
| 触发器      | 会触发 `DELETE` 触发器                                                    | 不会触发 `DELETE` 触发器，但可能会触发 `TRUNCATE` 触发器 (取决于数据库系统) |
| 外键约束    | 如果存在外键约束，则需要先删除或更新相关表中的数据，才能删除主表中的数据 | 如果存在外键约束，则可能无法使用 `TRUNCATE` 语句                             |

**总结:**

* 使用 `DELETE` 语句可以删除表中的指定行，而 `TRUNCATE` 语句会删除表中的所有行。
* `TRUNCATE` 语句的执行速度通常比 `DELETE` 语句快，因为它不会记录日志。
* `DELETE` 语句的操作可以回滚，而 `TRUNCATE` 语句的操作不能回滚。


## 67. DROP 和 TRUNCATE 的区别是什么？

`DROP` 和 `TRUNCATE` 都是用于操作数据库表的 SQL 语句，但它们的功能完全不同：

| 特性    | DROP                                                           | TRUNCATE                                                       |
|-----------|-----------------------------------------------------------------|----------------------------------------------------------------|
| 功能      | 从数据库中完全删除表，包括表结构、数据、索引、约束和权限等       | 只删除表中的所有数据，保留表结构、索引和约束等                 |
| 速度      | 相对较慢，因为它需要删除和释放更多的数据库对象                 | 相对较快，因为它只需要删除数据页                             |
| 可恢复性  | 不可恢复，一旦执行 `DROP` 语句，表将被永久删除                 | 数据可以恢复 (取决于数据库系统的事务日志机制)                |
| 事务      | 是一个 DDL 语句，操作不能回滚                                 | 是一个 DDL 语句，操作不能回滚                                 |

**总结:**

* `DROP` 语句用于完全删除数据库表及其所有相关对象，而 `TRUNCATE` 语句只删除表中的数据，保留表结构。
* `DROP` 语句的操作不可恢复，而 `TRUNCATE` 语句的操作在某些情况下可以恢复。


## 68. HAVING 和 WHERE 语句有什么区别？

`HAVING` 和 `WHERE` 都是用于过滤数据的 SQL 语句，但它们应用于查询的不同阶段：

| 特性       | WHERE                                                                                                       | HAVING                                                                                                       |
|--------------|------------------------------------------------------------------------------------------------------------|------------------------------------------------------------------------------------------------------------|
| 应用阶段     | 在分组之前，用于过滤行                                                                                       | 在分组之后，用于过滤组                                                                                       |
| 数据类型     | 可以使用任何数据类型                                                                                           | 只能使用聚合函数的结果或在 `GROUP BY` 子句中指定的列                                                       |
| 执行顺序     | 在 `GROUP BY` 子句之前执行                                                                                 | 在 `GROUP BY` 子句之后执行                                                                                 |
| 示例        | `SELECT * FROM employees WHERE department_id = 1;`                                                       | `SELECT department_id, AVG(salary) FROM employees GROUP BY department_id HAVING AVG(salary) > 50000;` |

**总结:**

* `WHERE` 子句用于过滤行，在分组之前应用；而 `HAVING` 子句用于过滤组，在分组之后应用。
* `WHERE` 子句可以使用任何数据类型，而 `HAVING` 子句只能使用聚合函数的结果或在 `GROUP BY` 子句中指定的列。


## 69. 如何向表中添加记录？

可以使用 `INSERT INTO` 语句向表中添加记录。`INSERT INTO` 语句的基本语法如下:

```sql
INSERT INTO table_name (column1, column2, ...) VALUES (value1, value2, ...);
```

* **`INSERT INTO`**:  关键字，表示要插入数据。
* **`table_name`**:  要插入数据的表的名称。
* **`(column1, column2, ...)`**:  可选的，要插入数据的列的名称列表。 如果省略，则默认插入所有列。
* **`VALUES`**:  关键字，表示要插入的值。
* **`(value1, value2, ...)`**:  要插入的值的列表，值的数量和顺序必须与列列表一致。

**示例:**

```sql
-- 向 employees 表中插入一条记录
INSERT INTO employees (employee_id, first_name, last_name, department_id) 
VALUES (100, 'John', 'Doe', 1);
```


## 70. 如何删除表中的记录？

可以使用 `DELETE` 语句从表中删除记录。`DELETE` 语句的基本语法如下:

```sql
DELETE FROM table_name WHERE condition;
```

* **`DELETE FROM`**:  关键字，表示要删除数据。
* **`table_name`**:  要删除数据的表的名称。
* **`WHERE condition`**:  可选的，指定要删除哪些行的条件。 如果省略，则删除表中的所有行。

**示例:**

```sql
-- 删除 employees 表中 department_id 为 1 的所有记录
DELETE FROM employees WHERE department_id = 1;
```

**注意:**

* 使用 `DELETE` 语句删除数据时要小心，因为它会永久删除数据。
* 在执行 `DELETE` 语句之前，最好先备份数据或使用事务来确保操作的安全性。
