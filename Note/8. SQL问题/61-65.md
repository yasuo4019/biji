## 61. SQL 中的非范式化 (Denormalization) 是什么，为什么要使用非范式化？

非范式化 (Denormalization) 是指有意地在数据库设计中引入数据冗余，与数据库范式化的目标相反。 它通常是通过将数据从多个表合并到一个表中来实现的，即使这会导致数据重复。

**为什么要使用非范式化？**

* **提高查询性能**:  非范式化可以减少数据库连接操作的数量，从而提高查询性能。 当数据库需要处理大量读操作而写操作相对较少时，这一点尤为重要。
* **简化查询**: 非范式化可以将来自多个表的数据合并到一个表中，从而简化查询语句，提高代码可读性。

**非范式化的缺点:**

* **数据冗余**: 非范式化会导致数据冗余，增加存储空间的使用量。
* **数据不一致**: 数据冗余可能会导致数据不一致，因为对数据的更新需要在多个位置进行。

**总结:**

非范式化是一种权衡，需要根据具体的应用场景来决定是否使用。 如果查询性能是首要考虑因素，并且数据一致性可以通过其他方式来保证，那么非范式化可能是一个不错的选择。


## 62. 重命名列与列的别名有什么区别？

**重命名列** 和 **列的别名** 都可以用来改变查询结果中列的显示名称，但它们之间存在一些关键区别：

| 特性        | 重命名列 (`ALTER TABLE ... RENAME COLUMN`) | 列的别名 (`AS`) |
|--------------|------------------------------------------|--------------------|
| **作用范围**    | 永久更改表结构                         | 仅限当前查询       |
| **语法位置**    | 在 `ALTER TABLE` 语句中使用              | 在 `SELECT` 语句中使用 |
| **示例**       | `ALTER TABLE employees RENAME COLUMN name TO full_name;` | `SELECT name AS full_name FROM employees;` |

**总结:**

* 使用 `ALTER TABLE ... RENAME COLUMN` 语句重命名列会永久更改数据库表结构，而使用 `AS` 关键字为列指定别名只会在当前查询结果中生效。
* 重命名列通常用于数据库设计阶段或需要永久更改列名的情况下，而列的别名通常用于简化查询语句或使查询结果更易于理解。


## 63. 嵌套子查询和相关子查询 (Correlated Subquery) 有什么区别？

**嵌套子查询** 和 **相关子查询** 都是嵌入在另一个查询 (外部查询) 中的查询 (内部查询)。 它们的主要区别在于子查询是否依赖于外部查询：

* **嵌套子查询 (Nested Subquery)**:  
    * 子查询可以独立于外部查询执行，因为它不引用外部查询中的任何列。
    * 子查询的结果通常作为外部查询的条件使用。
    * 例如: `SELECT * FROM employees WHERE department_id IN (SELECT id FROM departments WHERE location = 'New York');`

* **相关子查询 (Correlated Subquery)**:
    * 子查询依赖于外部查询，因为它引用了外部查询中的列。
    * 子查询会为外部查询中的每一行执行一次。
    * 例如: `SELECT * FROM employees e WHERE salary > (SELECT AVG(salary) FROM employees WHERE department_id = e.department_id);`

**总结:**

* 嵌套子查询独立于外部查询，而相关子查询依赖于外部查询。
* 相关子查询通常用于比较外部查询中的每一行与子查询结果集的情况。


## 64. 聚合索引和非聚合索引的区别是什么？

**聚合索引 (Clustered Index)** 和 **非聚合索引 (Non-clustered Index)** 都是用于加速数据检索的数据库结构，但它们在数据存储和检索方式上有所不同：

| 特性         | 聚合索引                                                              | 非聚合索引                                                                                             |
|---------------|--------------------------------------------------------------------|-------------------------------------------------------------------------------------------------------------------|
| 数据存储     | 决定表数据的物理存储顺序                                                 | 不影响表数据的物理存储顺序，而是存储指向实际数据行的指针                                                       |
| 唯一性        | 每个表只能有一个聚合索引                                                     | 一个表可以有多个非聚合索引                                                                                         |
| 类比        | 字典 (按照字母顺序排列，可以快速找到单词)                             | 书的索引 (列出关键词和对应页码，可以快速找到相关内容，但不会改变书中内容的顺序)                               |

**总结:**

* 聚合索引决定了表数据的物理存储顺序，而
* 非聚合索引不影响表数据的物理存储顺序，而是存储指向实际数据行的指针。
* 聚合索引更适合用于范围查询和排序操作，而非聚合索引更适合用于查找特定值。


## 65. CASE() 函数是什么？

`CASE()` 函数是 SQL 中用于实现条件逻辑的语句，类似于其他编程语言中的 `if-then-else` 语句。 它允许你根据一个或多个条件的计算结果返回不同的值。

**语法:**

```sql
CASE
    WHEN condition1 THEN result1
    WHEN condition2 THEN result2
    ...
    ELSE resultN
END
```

* **CASE**:  标记 `CASE` 语句的开始。
* **WHEN condition THEN result**:  定义一个条件和对应的结果。 如果 `condition` 为真，则返回 `result`。
* **ELSE result**:  可选的。 如果所有 `WHEN` 条件都不满足，则返回 `ELSE` 子句中的 `result`。
* **END**:  标记 `CASE` 语句的结束。

**示例:**

```sql
-- 根据员工的职位返回不同的工资等级
SELECT 
    employee_name,
    job_title,
    CASE
        WHEN job_title = 'Manager' THEN 'High'
        WHEN job_title LIKE '%Senior%' THEN 'Mid'
        ELSE 'Low'
    END AS salary_level
FROM
    employees;
```

**注意:**

* `CASE` 语句可以嵌套在其他 SQL 语句中，例如 `SELECT`, `UPDATE`, `DELETE` 等。
* `CASE` 语句的返回值可以是任何数据类型，例如数字、字符串、日期等。
