## 46. 什么是字符操作函数？举一些例子。

字符操作函数是 SQL 中用于处理和操作字符串数据的函数。 它们可以执行各种操作，例如连接字符串、提取子字符串、替换字符、查找子字符串的位置以及修剪字符串。

以下是一些常用的字符操作函数：

* **`CONCAT()`**: 将两个或多个字符串连接成一个字符串。 
    * 例如，`CONCAT('Hello', ' ', 'World')`  返回 "Hello World"。
* **`SUBSTR()` (或 `SUBSTRING()`)**: 从字符串中提取指定位置和长度的子字符串。
    * 例如，`SUBSTR('HelloWorld', 6, 5)` 返回 "World"。
* **`LENGTH()` (或 `LEN()`)**: 返回字符串的长度（字符数）。
    * 例如，`LENGTH('HelloWorld')` 返回 10。
* **`REPLACE()`**: 将字符串中所有出现的指定子字符串替换为另一个子字符串。
    * 例如，`REPLACE('Hello World', 'World', 'Everyone')` 返回 "Hello Everyone"。
* **`INSTR()` (或 `POSITION()`)**: 返回指定子字符串在字符串中第一次出现的位置（从 1 开始）。如果未找到子字符串，则返回 0。
    * 例如，`INSTR('HelloWorld', 'World')` 返回 7。
* **`LPAD()`**: 在字符串左侧填充指定字符，使其达到指定的长度。
    * 例如，`LPAD('Hello', 10, '*')` 返回 "*****Hello"。
* **`RPAD()`**: 在字符串右侧填充指定字符，使其达到指定的长度。
    * 例如，`RPAD('Hello', 10, '*')` 返回 "Hello*****"。
* **`TRIM()`**:  从字符串开头和结尾删除指定的字符（默认为空格）。
    * 例如，`TRIM('  Hello World  ')` 返回 "Hello World"。


## 47. 局部变量和全局变量有什么区别？

局部变量和全局变量的主要区别在于它们的作用域和生命周期：

**局部变量:**

* **作用域**: 仅限于声明它的函数或代码块内部。在函数外部或其他代码块中无法访问。
* **生命周期**: 从声明变量开始到函数或代码块执行完毕为止。函数执行完毕后，局部变量的内存空间会被释放。

**全局变量:**

* **作用域**: 在整个程序中都可以访问，包括函数内部。
* **生命周期**: 从程序开始运行到程序结束为止。

**示例:**

```sql
-- 定义一个全局变量
DECLARE global_var INT DEFAULT 10;

CREATE FUNCTION MyFunction()
RETURNS INT
BEGIN
    -- 定义一个局部变量
    DECLARE local_var INT DEFAULT 5;
    
    -- 修改全局变量和局部变量的值
    SET global_var = global_var + 1;
    SET local_var = local_var * 2;
    
    -- 返回局部变量的值
    RETURN local_var;
END;

-- 调用函数
SELECT MyFunction();  -- 返回 10

-- 查看全局变量的值
SELECT global_var;   -- 返回 11
```

在上面的例子中：

* `global_var` 是一个全局变量，可以在函数内部和外部访问。
* `local_var` 是函数 `MyFunction()` 中的局部变量，只能在函数内部访问。
* 当调用 `MyFunction()` 函数时，会修改全局变量 `global_var` 和局部变量 `local_var` 的值。
* 函数执行完毕后，局部变量 `local_var` 不再可用，而全局变量 `global_var` 的值仍然保留。


## 48. `ORDER BY` 语句的默认排序是什么？如何更改？

`ORDER BY` 语句用于对查询结果集进行排序。默认情况下，`ORDER BY` 语句使用**升序 (ASC)** 排序，即从小到大或从 A 到 Z。

要更改排序顺序为**降序 (DESC)**，只需在列名后面添加 `DESC` 关键字即可。

**示例:**

```sql
-- 默认升序排序
SELECT * FROM Customers ORDER BY Country; 

-- 降序排序
SELECT * FROM Customers ORDER BY Country DESC; 
```


## 49. 你知道哪些集合运算符？

集合运算符用于组合两个或多个 SELECT 语句的结果集，并返回一个新的结果集。

以下是常用的集合运算符：

* **`UNION`**: 返回两个 SELECT 语句的结果集的并集，**去除重复行**。
* **`UNION ALL`**: 返回两个 SELECT 语句的结果集的并集，**保留所有行**，包括重复行。
* **`INTERSECT`**: 返回两个 SELECT 语句的结果集的交集，即**同时出现在两个结果集中的行**。
* **`EXCEPT` (或 `MINUS`)**: 返回第一个 SELECT 语句的结果集中**存在于第一个结果集中但不存在于第二个结果集中的行**。 

**注意:** 

* 使用集合运算符时，所有 SELECT 语句必须具有相同的列数和相同的数据类型。
* `INTERSECT` 和 `EXCEPT` 运算符在某些数据库系统中可能不支持，例如 MySQL。


## 50. 查询中使用什么运算符进行模式匹配？

在 SQL 查询中，可以使用 `LIKE` 运算符进行模式匹配。`LIKE` 运算符通常与通配符一起使用，以匹配部分字符串。

常用的通配符包括：

* **`%`**: 匹配任意数量的字符（包括零个字符）。
* **`_`**: 匹配单个字符。

**示例:**

* `WHERE name LIKE 'A%'`: 匹配以字母 "A" 开头的所有姓名。
* `WHERE name LIKE '%son'`: 匹配以 "son" 结尾的所有姓名。
* `WHERE name LIKE 'a_e'`: 匹配以 "a" 开头，以 "e" 结尾，中间只有一个字符的所有姓名。
* `WHERE name LIKE '%a%'`: 匹配包含字母 "a" 的所有姓名。
