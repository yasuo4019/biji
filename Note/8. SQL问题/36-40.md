## 36. 如何从两个表中选取共同的记录？

可以使用以下几种方法从两个表中选取共同的记录：

* **`INTERSECT` 运算符:** (部分数据库系统支持)

   ```sql
SELECT column1, column2, ...
FROM table_1
INTERSECT
SELECT column1, column2, ...
FROM table_2;
   ```

* **`JOIN` 语句:**

   ```sql
SELECT DISTINCT column1, column2, ...
FROM table_1
INNER JOIN table_2 ON table_1.common_column = table_2.common_column;
   ```

**解释:**

* `INTERSECT` 运算符返回两个查询结果中都存在的记录。
* `JOIN` 语句通过连接条件将两个表连接起来，`INNER JOIN` 只返回满足连接条件的记录，使用 `DISTINCT` 关键字可以去除重复记录。

**注意:**

* `INTERSECT` 运算符的效率可能低于 `JOIN` 语句，尤其是在处理大型表时。
* 并非所有数据库系统都支持 `INTERSECT` 运算符。


## 37. 什么是 `DISTINCT` 语句，如何使用？

`DISTINCT` 语句用于去除查询结果中的重复记录，只返回唯一值。

**语法:**

```sql
SELECT DISTINCT column1, column2, ...
FROM table_name;
```

**解释:**

* `DISTINCT` 关键字用于指定只返回唯一值。
* `column1, column2, ...`：要查询的列名，可以指定一个或多个列。

**示例:**

```sql
-- 查询 customers 表中所有不重复的 customer_city 值
SELECT DISTINCT customer_city
FROM customers;
```


## 38. 什么是实体？举一些例子。

在关系型数据库中，**实体**是指现实世界中可以被明确区分的事物或概念，它可以是物理存在的，也可以是抽象的概念。每个实体都具有一些属性来描述其特征。

**实体的例子:**

* **顾客:**  属性可以包括顾客ID、姓名、地址、电话号码等。
* **产品:** 属性可以包括产品ID、名称、描述、价格等。
* **订单:** 属性可以包括订单ID、顾客ID、下单日期、总金额等。
* **书籍:**  属性可以包括书籍ID、书名、作者、出版社、出版日期等。


## 39. 什么是关系？举一些例子。

**关系**是指数据库中不同实体之间的关联方式。在关系型数据库中，关系通过表之间的外键约束来实现。

**关系的例子:**

* **顾客** 和 **订单:** 一个顾客可以有多个订单，一个订单只属于一个顾客。这种关系称为一对多关系 (1:N)。
* **学生** 和 **课程:** 一个学生可以选择多门课程，一门课程可以有多个学生选择。这种关系称为多对多关系 (M:N)。
* **作者** 和 **书籍:**  一本书可以有多个作者，一个作者可以写多本书。这种关系也是多对多关系 (M:N)。


## 40. 什么是 `NULL`？它与 0 或空格有什么不同？

在数据库中，`NULL` 表示**缺失值**或**未知值**，它与 0 或空格不同。

* **`NULL`:**  表示数据不存在或未知。
* **0:** 表示一个数值，是有意义的。
* **空格:** 表示一个长度为 0 的字符串，也是有意义的。

**示例:**

假设有一个学生信息表，其中包含姓名、年龄和电话号码等列。

* 如果某个学生的电话号码未知，则该列的值应为 `NULL`。
* 如果某个学生的年龄为 0，则表示该学生刚出生。
* 如果某个学生的姓名列为空格，则表示该学生的姓名为空字符串。


**总结:**

`NULL`、0 和空格代表不同的含义，在数据库设计和数据分析时要区分对待。
