
递归查询是一种自引用的查询方式，常用于处理树形结构的数据，如组织架构、目录结构等。

---

递归查询使用WITH RECURSIVE子句，其基本语法如下：

WITH RECURSIVE cte_name AS ( 
	初始查询 
	UNION ALL 
	递归查询 
) 
SELECT * FROM cte_name;

---

我们可以使用 `employees` 表来展示一个递归查询的示例。假设我们在表中添加一个 `manager_id` 列，用于表示每个员工的直接上级。

### 更新后的 `employees` 表结构：

| id | name  | department | salary | manager_id |
|----|-------|------------|--------|------------|
| 1  | Alice | Sales      | 50000  | NULL       |
| 2  | Bob   | Sales      | 60000  | 1          |
| 3  | Carol | HR         | 55000  | 1          |
| 4  | Dave  | HR         | 70000  | 3          |
| 5  | Eve   | Sales      | 50000  | 2          |

### 递归查询示例：

我们想要查找某个员工（如 Alice）及其所有下属。可以使用以下 SQL 查询：

```sql
WITH RECURSIVE EmployeeHierarchy AS (
    SELECT id, name, manager_id
    FROM employees
    WHERE name = 'Alice'  -- 起始条件

    UNION ALL

    SELECT e.id, e.name, e.manager_id
    FROM employees e
    INNER JOIN EmployeeHierarchy eh ON e.manager_id = eh.id
)

SELECT * FROM EmployeeHierarchy;
```

### 结果将是：

| id | name  | manager_id |
|----|-------|------------|
| 1  | Alice | NULL       |
| 2  | Bob   | 1          |
| 3  | Carol | 1          |
| 5  | Eve   | 2          |
| 4  | Dave  | 3          |

### 说明：
1. **初始查询**：选择名为 Alice 的员工。
2. **递归部分**：通过 `INNER JOIN` 连接下属员工，查找所有直接或间接报告给 Alice 的员工。
3. **结果**：最终返回 Alice 及其所有下属的 ID 和姓名。
