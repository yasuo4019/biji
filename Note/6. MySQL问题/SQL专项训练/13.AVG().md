`AVG()` 是 SQL 中的一个聚合函数，用于计算指定列的平均值。它通常用于数值数据，帮助分析数据集的整体趋势。

### 1. 基本用法

#### 计算平均值

使用 `AVG(column_name)` 可以计算指定列的平均值。

#### 示例
```sql
SELECT AVG(salary) AS AverageSalary
FROM employees;
```
这个查询返回 `employees` 表中所有员工薪资的平均值，并将结果列命名为 `AverageSalary`。

### 2. 与 `WHERE` 子句结合使用

可以使用 `WHERE` 子句来过滤数据，仅计算符合条件的行的平均值。

#### 示例
```sql
SELECT AVG(salary) AS AverageHighSalaries
FROM employees
WHERE salary > 50000;
```
这个查询返回薪资大于 50,000 的员工的平均薪资，并将结果列命名为 `AverageHighSalaries`。

### 3. 忽略 NULL 值

`AVG()` 函数会自动忽略 NULL 值，因此在计算时只会考虑非 NULL 的数值。

### 4. 与 `GROUP BY` 结合使用

结合使用 `AVG()` 和 `GROUP BY` 可以计算每个分组的平均值。

#### 示例
```sql
SELECT department, AVG(salary) AS AverageSalary
FROM employees
GROUP BY department;
```
这个查询返回每个部门的员工薪资平均值，并将结果列命名为 `AverageSalary`。

### 5. 使用别名

可以使用 `AS` 关键字为计算结果指定别名，以提高结果的可读性。

#### 示例
```sql
SELECT AVG(salary) AS [Average Employee Salary]
FROM employees;
```
这个查询返回所有员工薪资的平均值，并将结果列命名为 `Average Employee Salary`。

### 总结

- **`AVG(column_name)`**: 计算指定列的平均值。
- **条件计算**: 可以使用 `WHERE` 子句过滤数据。
- **自动忽略 NULL**: `AVG()` 会忽略 NULL 值。
- **结合 `GROUP BY`**: 计算每个分组的平均值。
- **使用别名**: 提高结果的可读性。

`AVG()` 函数在数据分析和决策支持中非常有用，能够帮助理解数据的中心趋势。
