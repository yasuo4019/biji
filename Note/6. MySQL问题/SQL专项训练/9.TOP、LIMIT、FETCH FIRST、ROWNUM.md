在 SQL 中，`TOP`、`LIMIT`、`FETCH FIRST` 和 `ROWNUM` 子句用于限制查询结果集中的行数。不同的数据库系统使用不同的语法来实现这一功能。

### 1. `TOP` 子句

- **数据库**: SQL Server, MS Access
- **功能**: 返回结果集中的前 N 行。

#### 示例

```sql
SELECT TOP 5 * FROM employees;
```
这个查询返回 `employees` 表中的前 5 行记录。

### 2. `LIMIT` 子句

- **数据库**: MySQL, PostgreSQL, SQLite
- **功能**: 限制结果集中的行数。

#### 示例

```sql
SELECT * FROM employees LIMIT 5;
```
这个查询返回 `employees` 表中的前 5 行记录。

### 3. `FETCH FIRST` 子句

- **数据库**: Oracle (12c 及以上), DB2, SQL Server
- **功能**: 返回结果集中的前 N 行。

#### 示例

```sql
SELECT * FROM employees FETCH FIRST 5 ROWS ONLY;
```
这个查询返回 `employees` 表中的前 5 行记录。

### 4. `ROWNUM` 子句

- **数据库**: Oracle
- **功能**: 生成结果集中的行号，可以用来限制结果。

#### 示例

```sql
SELECT * FROM employees WHERE ROWNUM <= 5;
```
这个查询返回 `employees` 表中的前 5 行记录。

### 组合使用

在某些情况下，可以与 `ORDER BY` 子句组合使用，以确保返回特定排序的前 N 行。

**示例**（以 PostgreSQL 为例）:
```sql
SELECT * FROM employees ORDER BY age DESC LIMIT 5;
```
这个查询返回年龄最大的前 5 位员工。

### 总结

- **TOP**: 用于 SQL Server。
- **LIMIT**: 用于 MySQL 和 PostgreSQL。
- **FETCH FIRST**: 用于 Oracle 12c 及以上版本和其他数据库。
- **ROWNUM**: 特有于 Oracle，用于生成行号。

根据使用的数据库系统选择合适的语法来限制查询结果。
