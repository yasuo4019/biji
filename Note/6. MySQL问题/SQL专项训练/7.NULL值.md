在 SQL 中，`NULL` 值表示缺失或未知的值。它并不等同于零或空字符串，而是一个特殊的标记，表示数据的缺失。

### 1. NULL 的特点

- **表示缺失**: `NULL` 用于表示一列中没有可用的数据。
- **数据类型**: `NULL` 可以出现在任何数据类型的列中，比如整数、字符串、日期等。

### 2. NULL 的使用

#### 1. 插入 NULL 值

在插入数据时，可以将某列设置为 `NULL`。

**示例**:
```sql
INSERT INTO employees (name, age, department)
VALUES ('Alice', NULL, 'Sales');
```
这个查询将 Alice 的年龄设置为 `NULL`。

#### 2. 查询 NULL 值

使用 `IS NULL` 或 `IS NOT NULL` 来检查 `NULL` 值。

**示例**:
```sql
SELECT * FROM employees
WHERE age IS NULL;
```
这个查询返回所有年龄为 `NULL` 的员工。

```sql
SELECT * FROM employees
WHERE age IS NOT NULL;
```
这个查询返回所有年龄不为 `NULL` 的员工。

### 3. NULL 与比较运算符

- `NULL` 不等于任何值，包括它自身。
- 使用普通的比较运算符（如 `=`, `!=`）无法正确比较 `NULL` 值。

**示例**:
```sql
SELECT * FROM employees
WHERE age = NULL;  -- 这将不会返回任何结果
```

### 4. 使用 COALESCE 函数

`COALESCE` 函数可以用来处理 `NULL` 值，返回第一个非 NULL 的值。

**示例**:
```sql
SELECT name, COALESCE(age, 'Unknown') AS age
FROM employees;
```
这个查询返回每个员工的名字和年龄，如果年龄为 `NULL`，则显示为 'Unknown'。

### 5. NULL 值的排序

在排序时，`NULL` 值通常被视为最小或最大值，具体行为可能依赖于数据库系统。

**示例**:
```sql
SELECT * FROM employees
ORDER BY age;
```
在这个查询中，`NULL` 值可能会出现在结果的最前面或最后面，具体取决于数据库的实现。

### 总结

- `NULL` 表示缺失数据，不能直接与其他值进行比较。
- 使用 `IS NULL` 和 `IS NOT NULL` 来检查 `NULL` 值。
- `COALESCE` 函数可用于处理和替代 `NULL` 值。
