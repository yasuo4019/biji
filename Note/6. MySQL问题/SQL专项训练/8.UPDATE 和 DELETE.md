在 SQL 中，`UPDATE` 和 `DELETE` 语句用于修改和删除数据库中的记录。它们是数据操作语言（DML）的重要组成部分。

### 1. `UPDATE` 语句

`UPDATE` 语句用于修改现有记录的值。

#### 基本语法

```sql
UPDATE table_name
SET column1 = value1, column2 = value2
WHERE condition;
```

#### 示例

假设有一个名为 `employees` 的表，包含以下字段：`id`, `name`, `age`, `department`。

- **更新单条记录**:
```sql
UPDATE employees
SET age = 31
WHERE name = 'Alice';
```
这个查询将名为 Alice 的员工的年龄更新为 31。

- **更新多条记录**:
```sql
UPDATE employees
SET department = 'Marketing'
WHERE age < 30;
```
这个查询将所有年龄小于 30 岁的员工的部门更新为 Marketing。

### 2. `DELETE` 语句

`DELETE` 语句用于删除符合条件的记录。

#### 基本语法

```sql
DELETE FROM table_name
WHERE condition;
```

#### 示例

- **删除单条记录**:
```sql
DELETE FROM employees
WHERE name = 'Bob';
```
这个查询将删除名为 Bob 的员工记录。

- **删除多条记录**:
```sql
DELETE FROM employees
WHERE age > 60;
```
这个查询将删除所有年龄大于 60 岁的员工记录。

### 注意事项

- **使用 WHERE 子句**: 
  - 在 `UPDATE` 和 `DELETE` 语句中，`WHERE` 子句是非常重要的。没有 `WHERE` 子句，所有记录都会被更新或删除。
  
  **示例**:
  ```sql
  DELETE FROM employees;  -- 这将删除所有员工记录
  ```

- **备份数据**: 在执行 `UPDATE` 和 `DELETE` 操作前，建议备份数据，尤其是在生产环境中。

- **事务控制**: 在支持事务的数据库中，可以使用 `BEGIN`, `COMMIT`, 和 `ROLLBACK` 来管理数据更改。

### 总结

- `UPDATE` 用于修改现有记录，`DELETE` 用于删除记录。
- 使用 `WHERE` 子句来限制操作的范围，确保只影响所需的记录。
