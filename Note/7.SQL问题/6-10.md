## 6. 请给出一些常见 SQL 命令

* **DDL（数据定义语言）：** 用于定义数据库结构
    * `CREATE`：创建数据库或数据库对象（如表）。
    * `ALTER`：修改数据库对象的结构（如添加、修改或删除列）。
    * `TABLE`：与 `CREATE` 或 `ALTER` 一起使用，用于操作表。
    * `DROP`：删除数据库或数据库对象。
    * `TRUNCATE`：清空表中的数据，但保留表结构。
    * `ADD COLUMN`：向现有表添加列。
* **DML（数据操作语言）：** 用于操作数据库中的数据
    * `UPDATE`：修改表中的数据。
    * `DELETE`：删除表中的数据。
    * `INSERT`：向表中插入数据。
* **DCL（数据控制语言）：** 用于控制数据库的访问权限
    * `GRANT`：授予用户访问数据库的权限。
    * `REVOKE`：撤销用户的数据库访问权限。
* **TCL（事务控制语言）：** 用于管理数据库事务
    * `COMMIT`：提交事务，使更改永久生效。
    * `SET TRANSACTION`：设置事务的属性。
    * `ROLLBACK`：回滚事务，撤销已做的更改。
    * `SAVEPOINT`：在事务中设置保存点，以便回滚到特定点。
* **DQL（数据查询语言）：** 用于查询数据库中的数据
    * `SELECT`：从数据库中检索数据。

## 7. 什么是数据库？

数据库是一种组织化的数据集合，以一种结构化的方式存储数据，以便于访问、管理和更新。数据通常存储在表中，表由行和列组成，类似于电子表格。数据库可以存储各种类型的数据，例如文本、数字、图像和文件。

## 8. 什么是 DBMS？你知道哪些类型的 DBMS？

DBMS 是数据库管理系统（Database Management System）的缩写。它是用于管理数据库的软件系统，提供以下功能：

* **数据定义：** 定义数据库和表结构。
* **数据操作：** 插入、更新、删除和查询数据。
* **数据安全：** 控制数据访问和保护数据完整性。
* **数据恢复：** 从故障中恢复数据。

**DBMS 的类型：**

* **关系型数据库管理系统 (RDBMS)：**  数据以表格形式存储，并通过关系（外键）连接。例如：MySQL, PostgreSQL, Oracle, SQL Server。
* **层次型数据库管理系统：** 数据以树状结构存储，类似于文件系统。例如：IMS (Information Management System)。
* **网络型数据库管理系统：** 数据以图状结构存储，允许更复杂的 relationship。例如：IDMS (Integrated Database Management System)。
* **面向对象数据库管理系统 (OODBMS)：** 数据以对象的形式存储，类似于面向对象编程语言。例如：ObjectDB, db4o。
* **NoSQL 数据库：** 不遵循传统的关系型数据库模型，例如文档数据库、键值存储等。例如：MongoDB, Cassandra, Redis。

## 9. 什么是关系型数据库管理系统？举一些 RDBMS 的例子。

关系型数据库管理系统 (RDBMS) 是一种使用关系模型来管理数据的数据库管理系统。关系模型使用表格来组织数据，表格之间通过关系（外键）连接。

**RDBMS 的特点：**

* 数据以表格形式存储。
* 表格之间通过关系关联。
* 使用结构化查询语言 (SQL) 进行数据操作。

**RDBMS 的例子：**

* MySQL
* PostgreSQL
* Oracle
* SQL Server
* MariaDB

## 10. SQL 中的表和字段是什么？

* **表 (Table)：** 表是数据库中存储数据的基本单元。它以表格形式组织数据，由行和列组成。每行代表一个数据记录，每列代表一个数据属性。
* **字段 (Field)：** 字段是表中的一列，也称为列名。每个字段代表一个特定的数据属性，例如姓名、年龄、地址等。每个字段都有一个数据类型，例如整数、字符串、日期等。
