## 11. 什么是 SQL 查询，你知道哪些类型的查询？

SQL 查询是用 SQL 语言编写的语句，用于从数据库中检索或修改数据。

SQL 查询主要分为以下两类：

* **数据查询语言 (DQL)：** 用于检索数据。最常见的 DQL 语句是 `SELECT`，它允许用户指定要检索的列、要从中检索数据的表以及任何筛选条件。 DQL 还包括用于限制、排序和分组结果的子句，以及用于从多个表中联接数据的子句。
* **数据操作语言 (DML)：** 用于修改数据。主要的 DML 语句包括：
    * `INSERT`：向表中添加新数据。
    * `UPDATE`：修改现有数据。
    * `DELETE`：从表中删除数据。

## 12. 什么是子查询？

子查询，也称为内部查询，是嵌入在另一个 SQL 查询（称为外部查询）中的查询。子查询可以出现在 `SELECT`、`FROM`、`WHERE` 和 `UPDATE` 等子句中。 

子查询的工作原理是先执行最内层的子查询，并将结果传递给外部查询。外部查询再根据子查询的结果进行进一步的操作。

## 13. 你知道哪些类型的 SQL 子查询？

* **单行子查询：** 最多返回一行数据。通常与比较运算符（例如 `=`、`>`、`<`）一起使用。
* **多行子查询：** 返回多行数据。通常与集合运算符（例如 `IN`、`ANY`、`ALL`）或比较运算符（配合 `EXISTS` 或 `NOT EXISTS`）一起使用。
* **多列子查询：** 返回多列数据。通常用于需要比较多个列值的情况。
* **相关子查询：** 引用外部查询中列的子查询。这意味着子查询的执行依赖于外部查询的值。
* **嵌套子查询：** 子查询中嵌套了其他子查询。执行顺序是从最内层到最外层。

## 14. 什么是约束，为什么使用约束？

约束是数据库表中对列数据的限制条件，用于确保数据的完整性和有效性。

**使用约束的原因：**

* **数据完整性：** 确保数据库中的数据准确、一致且可靠。
* **数据有效性：** 限制输入的数据类型、格式和范围，防止无效数据的插入。
* **数据一致性：** 维持不同表之间数据的一致性，例如使用外键约束。
* **防止数据丢失：** 通过限制删除操作，防止意外删除重要数据。

## 15. 你知道哪些 SQL 约束？

* **`DEFAULT`：** 为列提供默认值。如果未插入值，则使用默认值。
* **`UNIQUE`：** 确保列中的每个值都是唯一的。
* **`NOT NULL`：** 确保列中不允许空值。
* **`PRIMARY KEY`：** 唯一标识表中的每一行。它必须包含唯一值，并且不能为空。
* **`FOREIGN KEY`：** 用于建立两个表之间的关系。它确保一个表中的列（子表）的值必须与另一个表中相应列（父表）的值匹配。
