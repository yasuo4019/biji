## 51. 主键和唯一键有什么区别？

主键和唯一键都是用于保证数据表中列的唯一性的约束，但它们之间有一些关键区别：

**主键 (Primary Key):**

* **唯一性**:  主键列中的值必须唯一，不允许重复值，也不允许 NULL 值。
* **标识性**:  主键用于唯一标识表中的每一行记录，类似于身份证号码。
* **每个表只能有一个主键**: 一个表只能定义一个主键约束。
* **通常作为外键关联**: 主键通常被其他表的外键引用，用于建立表之间的关系。

**唯一键 (Unique Key):**

* **唯一性**: 唯一键列中的值也必须唯一，不允许重复值，但可以包含 NULL 值（并且可以有多个 NULL 值，因为 NULL 不等于 NULL）。
* **标识性**: 唯一键主要用于保证数据的唯一性，而不是用于标识记录。
* **一个表可以有多个唯一键**: 一个表可以定义多个唯一键约束。
* **不一定作为外键关联**: 唯一键可以作为外键被其他表引用，但不是必须的。

**总结:**

* 主键用于确保每行数据的唯一标识，而唯一键用于防止数据重复。
* 一个表只能有一个主键，但可以有多个唯一键。
* 主键不能为空，而唯一键可以为空。


## 52. 什么是复合主键？

复合主键是指由**两个或多个列**组合而成的主键。 当单个列无法唯一标识表中的每一行记录时，可以使用复合主键。

**示例:**

假设有一个名为 `OrderDetails` 的表，用于存储订单的详细信息，包含以下列：

* `OrderID`: 订单ID
* `ProductID`: 产品ID
* `Quantity`: 数量

在这个表中，单个列 `OrderID` 或 `ProductID` 都无法唯一标识一行记录，因为一个订单可以包含多个产品。 因此，可以使用 `OrderID` 和 `ProductID` 两列组合作为复合主键，以确保每行记录的唯一性。


## 53. SELECT 查询中常见语句的出现顺序是什么？

在 SQL 中，`SELECT` 查询语句的各个子句需要按照特定的顺序出现，否则会导致语法错误。 以下是常见的 `SELECT` 查询语句的子句顺序：

1. `SELECT`
2. `FROM`
3. `JOIN`
4. `ON`
5. `WHERE`
6. `GROUP BY`
7. `HAVING`
8. `ORDER BY`
9. `LIMIT` (或 `OFFSET...FETCH`)

**示例:**

```sql
SELECT column1, column2, ...
FROM table1
JOIN table2 ON table1.column = table2.column
WHERE condition
GROUP BY column
HAVING condition
ORDER BY column ASC/DESC
LIMIT number;
```


## 54. 解释器执行 SELECT 查询语句的顺序是什么？

尽管 `SELECT` 查询语句的子句需要按照特定的顺序编写，但数据库引擎在执行查询时，并不会严格按照这个顺序。 实际上，数据库引擎会按照一定的逻辑顺序来执行查询，以优化查询性能。

以下是 SQL 解释器执行 `SELECT` 查询语句的典型顺序：

1. `FROM` 和 `JOIN`: 首先，确定要查询哪些表，并根据 `JOIN` 条件将它们连接起来。
2. `WHERE`: 然后，应用 `WHERE` 子句中的条件，过滤掉不符合条件的行。
3. `GROUP BY`: 接下来，根据 `GROUP BY` 子句中的列对结果集进行分组。
4. `HAVING`:  对分组后的结果集应用 `HAVING` 子句中的条件，过滤掉不符合条件的组。
5. `SELECT`:  确定要查询哪些列，并对这些列进行计算或操作。
6. `ORDER BY`:  根据 `ORDER BY` 子句中的列对结果集进行排序。
7. `LIMIT`: 最后，根据 `LIMIT` 子句限制返回的行数。

**注意:** 这只是一个典型的执行顺序，实际的执行顺序可能会因数据库系统、查询优化器和具体查询语句的不同而有所差异。


## 55. 什么是视图，为什么要使用视图？

视图 (View) 是基于一个或多个数据库表或其他视图的虚拟表。 视图本身不存储实际数据，它只是存储了查询定义。 当查询视图时，数据库引擎会动态地从底层表中检索数据。

**为什么要使用视图?**

视图提供以下优点:

* **简化复杂查询**: 可以将复杂的查询逻辑封装在视图中，简化查询语句，提高代码可读性。
* **增强数据安全性**: 可以通过视图限制用户对底层数据的访问权限，只允许用户查看或修改部分数据。
* **实现数据独立性**: 视图可以屏蔽底层数据结构的变化，例如列名或表名更改，从而提高应用程序的灵活性。
* **汇总数据**: 可以使用视图将多个表中的数据组合在一起，方便用户查看和分析。
* **节省存储空间**: 视图不存储实际数据，只存储查询定义，因此可以节省存储空间。

**示例:**

```sql
-- 创建一个名为 "ActiveCustomers" 的视图，
-- 用于显示所有状态为 "Active" 的客户信息
CREATE VIEW ActiveCustomers AS
SELECT customer_id, customer_name, email
FROM customers
WHERE status = 'Active';

-- 查询 "ActiveCustomers" 视图
SELECT * FROM ActiveCustomers;
```

在这个例子中，`ActiveCustomers` 视图只显示状态为 "Active" 的客户信息。 用户可以通过查询 `ActiveCustomers` 视图来获取这些信息，而无需编写复杂的 `WHERE` 子句。
