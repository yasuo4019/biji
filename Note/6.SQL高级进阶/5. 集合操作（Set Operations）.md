集合操作用于将两个或多个查询结果集进行合并或比较。常见的集合操作符包括UNION、 INTERSECT、EXCEPT。

---

基本语法如下： 
SELECT column_list FROM table1 
UNION [ALL] 
SELECT column_list FROM table2; 

SELECT column_list FROM table1 
INTERSECT 
SELECT column_list FROM table2; 

SELECT column_list FROM table1 
EXCEPT 
SELECT column_list FROM table2; 

---

### 例子：查找所有销售和人力资源部门的员工

假设我们有一个 `employees` 表，结构如下：

| id | name  | department | salary |
|----|-------|------------|--------|
| 1  | Alice | Sales      | 50000  |
| 2  | Bob   | Sales      | 60000  |
| 3  | Carol | HR         | 55000  |
| 4  | Dave  | HR         | 70000  |
| 5  | Eve   | Sales      | 50000  |

我们想要找到所有属于销售（Sales）和人力资源（HR）部门的员工，并将其合并为一个结果集。

### SQL 查询：

```sql
SELECT 
    id,
    name,
    department,
    salary
FROM 
    employees
WHERE 
    department = 'Sales'

UNION

SELECT 
    id,
    name,
    department,
    salary
FROM 
    employees
WHERE 
    department = 'HR';
```

### 结果将是：

| id | name  | department | salary |
|----|-------|------------|--------|
| 1  | Alice | Sales      | 50000  |
| 2  | Bob   | Sales      | 60000  |
| 5  | Eve   | Sales      | 50000  |
| 3  | Carol | HR         | 55000  |
| 4  | Dave  | HR         | 70000  |

### 说明：

1. **第一个查询**：
   - 从 `employees` 表中选择所有销售部门的员工。

2. **第二个查询**：
   - 从 `employees` 表中选择所有人力资源部门的员工。

3. **集合操作 `UNION`**：
   - `UNION` 将两个查询的结果合并，去除重复的记录（如果存在）。
