#### 1.联合索引在什么时候会失效？

##### 联合索引失效的情况
**答：** 联合索引在查询时遵循“最左前缀匹配原则”，即索引会按从左到右的顺序依次匹配联合索引中的列。如果在查询时没有按照联合索引中定义的顺序使用列，或者跳过了某一列，联合索引可能会部分失效或完全失效。

**具体解释：**
- 如果查询条件中使用了联合索引的第一个字段，索引是有效的。
- 如果查询条件直接从第二个字段开始，而跳过了第一个字段，索引将失效。
- 如果按照正确顺序查询，比如使用前两个字段，但忽略了第三个字段，索引仍然有效，但如果从中间某个字段开始，索引会失效。

**例子：**
假设你有一个联合索引 `(name, birthday, gender)`，查询时：
- `SELECT * FROM patients WHERE name = 'John';` — 联合索引有效。
- `SELECT * FROM patients WHERE name = 'John' AND birthday = '1990-01-01';` — 联合索引有效。
- `SELECT * FROM patients WHERE birthday = '1990-01-01';` — 联合索引失效。
- `SELECT * FROM patients WHERE birthday = '1990-01-01' AND gender = 'M';` — 联合索引失效。

#### 2.联合查询时，出现范围查询时的情况。

1.`select * from t_table where a > 1 and b = 2`中为什么字段b没有使用联合查询？

在数据库中使用联合索引时，最左匹配原则指的是索引从最左边的列开始依次匹配，直到遇到范围查询为止。原因如下：

1. **范围查询影响顺序扫描**: 当遇到范围查询（如 `>`、`<`、`BETWEEN` 等）时，索引无法继续有序地扫描后续列，因为范围查询会导致不确定性，无法按索引顺序高效定位后续列的具体值。

2. **索引结构的限制**: B树等索引结构在进行范围查询时，需要遍历多个节点。这种情况下，索引无法保持对后续列的精确匹配。

因此，一旦在联合索引中遇到范围查询，后续的列就不会被有效利用。为了优化查询，可以考虑调整索引顺序或使用其他索引策略。

---
2.`select * from t_table where a >= 1 and b = 2`中为什么a、b字段都使用了联合查询？

虽然 `a` 字段使用了 `>=` 进行范围查询，但联合索引仍然能够让 `b` 字段参与查询，原因如下：

1. **B+树结构**: 联合索引的 B+树结构允许在 `a` 字段满足 `>= 1` 的条件下，顺序地遍历接下来的记录。由于符合条件的记录是相邻的，数据库可以快速定位到满足 `a >= 1` 的第一条记录。

2. **有序性**: 在联合索引中，`a` 字段的值是有序的，而在 `a` 值相同的情况下，`b` 字段的值也是有序的。因此，当确定 `a` 字段的范围后，数据库可以在这个范围内使用 `b` 字段的条件进行高效过滤。

3. **减少扫描范围**: 在扫描 `a` 字段值为 1 的记录时，结合 `b = 2` 的条件，数据库可以直接从符合 `a = 1` 和 `b = 2` 的第一条记录开始扫描，而不必遍历所有 `a = 1` 的记录。这种方式显著减少了需要扫描的记录数量。

4. **执行计划的支持**: 从执行计划中看到的 `key_len` 为 8 字节，表明优化器确实使用了两个字段的查询条件，进一步验证了 `a` 和 `b` 字段都有效利用了联合索引。

因此，尽管 `a` 字段使用了范围查询，联合索引的设计允许 `b` 字段有效参与查询，优化了查询性能。

3.`SELECT * FROM t_table WHERE a BETWEEN 2 AND 8 AND b = 2`，联合索引 `(a, b)` 哪一个字段用到了联合索引的 B+Tree？

在这条查询语句中，**`a` 和 `b` 都使用了联合索引的 B+Tree** 进行索引查询。

### **为什么 `a` 和 `b` 字段都用到了联合索引？**

联合索引 `(a, b)` 是按字段 `a` 排在第一位、字段 `b` 排在第二位创建的，因此这个联合索引在 B+Tree 中会按以下顺序排列数据：

- 首先按照 `a` 的值排序。
- 如果 `a` 的值相同，再按 `b` 的值排序。

**具体分析：**

1. **`a` 字段**：在查询条件中，`a BETWEEN 2 AND 8` 使用了范围查询。因为 `a` 是联合索引的第一个字段，查询时可以有效利用 B+Tree 中的索引范围来找到满足 `a` 值在 2 到 8 之间的记录。

2. **`b` 字段**：虽然 `b` 在联合索引中是第二个字段，但因为联合索引已经根据 `a` 进行了范围限定，数据库可以在找到满足 `a` 值的记录后，继续利用联合索引在 B+Tree 中对 `b` 进行精确匹配查询。

### **为什么两个字段都能利用联合索引？**

- **最左前缀原则**：查询条件中的 `a` 字段使用了范围查询，因此首先利用了联合索引的第一个字段 `a`。接着，由于已经锁定了 `a` 的范围，联合索引能够继续使用 `b` 字段来进一步过滤符合条件的记录。

- **索引覆盖**：在查询过程中，数据库系统会利用联合索引中第一个字段 `a` 对结果进行范围过滤，然后根据 `b` 字段在 B+Tree 中对记录进行精确匹配。这使得联合索引 `(a, b)` 可以同时为 `a` 和 `b` 字段加速查询。

### **总结**
在这条查询语句中，`a` 字段通过联合索引的范围查找锁定了可能的记录范围，而 `b` 字段进一步利用联合索引的精确查找来缩小结果范围，从而提升查询效率。这就是为什么 `a` 和 `b` 都使用了联合索引的 B+Tree 进行索引查询。

---

综上所示，**联合索引的最左匹配原则，在遇到范围查询（如 >、<）的时候，就会停止匹配，也就是范围查询的字段可以用到联合索引，但是在范围查询字段的后面的字段无法用到联合索引。注意，对于 >=、<=、BETWEEN、like 前缀匹配的范围查询，并不会停止匹配，前面我也用了四个例子说明了**。
