
在 SQL 中，`ORDER BY` 子句用于对查询结果进行排序。它按照指定的列对结果进行升序（`ASC`）或降序（`DESC`）排列。

### **解释：**

在你的 SQL 查询中：
```sql
SELECT * FROM order WHERE status = 1 ORDER BY create_time ASC;
```

- **`ORDER BY create_time ASC`**：这部分指示数据库系统按照 `create_time` 列中的值对查询结果进行升序排序（从早到晚）。

### **详细说明：**

1. **`SELECT * FROM order WHERE status = 1`**：
   - 这部分查询从 `order` 表中选择所有 `status` 等于 1 的记录。

2. **`ORDER BY create_time ASC`**：
   - 查询结果根据 `create_time` 列的值进行排序。
   - `ASC` 表示升序（从小到大），所以结果会按照 `create_time` 的时间从早到晚排列。如果不指定 `ASC` 或 `DESC`，默认就是 `ASC`。

### **举例：**

假设 `order` 表中有以下数据：

| id  | status | create_time       |
|-----|--------|-------------------|
| 1   | 1      | 2023-08-01 10:00  |
| 2   | 1      | 2023-08-01 09:00  |
| 3   | 1      | 2023-08-01 11:00  |
| 4   | 2      | 2023-08-01 08:00  |

在执行这条查询后，返回的结果会是：

| id  | status | create_time       |
|-----|--------|-------------------|
| 2   | 1      | 2023-08-01 09:00  |
| 1   | 1      | 2023-08-01 10:00  |
| 3   | 1      | 2023-08-01 11:00  |

这些记录按照 `create_time` 的升序排序。
