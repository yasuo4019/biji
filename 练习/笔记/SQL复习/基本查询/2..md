## 1) 主数据层（客户/账户/机构/字典）

- `customer` 客户主数据（客户号、证件、状态）
- `account` 账户主数据（账号、客户、币种、账户类型、状态）
- `branch` 机构树（总行/分行/支行，自关联）
- `account_type_dict` / `txn_type_dict` 业务字典表（真实系统里很常见）

## 2) 交易与业务层（面向“请求/单据/流水”）

- `txn` 账户流水（展示/业务流水，幂等 request_id）
- `transfer_order` 转账业务主单（业务请求单）

> 这两个表分开非常关键：  
> **transfer_order 是“业务单”**，**txn 是“账户侧流水”**。现实系统里通常不会只靠一个表打天下。

## 3) 账务与余额层（账务事实 / 状态快照）

- `ledger_entry` 账务分录（借贷分录，接近总账/子账思想）
- `account_balance_snapshot` 余额快照（账面余额、可用余额、冻结金额）

> 这层是你从“会写查询”进阶到“理解金融系统”的关键。

## 4) 风控/控制层（冻结控制）

- `account_freeze_log` 冻结/解冻历史动作日志（历史事件）
- `account_freeze_hold` 当前冻结明细（当前生效态）

> 这对表设计非常真实：
> 
> - `log` = 历史动作
> - `hold` = 当前有效状态  
>    很多系统都会这样拆。

`transfer_order -> ledger_entry`

这一步会把你从“普通 SQL”拉到“金融业务 SQL”。

### 题1：查询所有客户及其账户信息（含账户状态中文化）

**业务意义**：客服/运营最常查。

**涉及表**：`customer`, `account`

**你要练到的点**：

- 内连接
- `CASE` 状态码转中文
- 排序

```sql
select
  c.customer_no,
  c.full_name,
  a.account_no,
  a.currency,
  a.account_type,
  case a.status
    when 1 then '正常'
    when 2 then '冻结'
    when 9 then '销户'
    else '未知'
  end as account_status,
  a.balance,
  a.created_at
from yewu.customer c
join yewu.account a
  on a.customer_id = c.customer_id
order by c.customer_no, a.account_no;
```

### 题2：查询“冻结账户”的客户信息及余额

**业务意义**：风控/客服排查冻结客户。

**涉及表**：`customer`, `account`, `account_balance_snapshot`

> 这里你会第一次感受到：
> 
> - `account.balance` 是演示字段
> - `account_balance_snapshot` 更接近真实状态余额

```sql

  c.customer_no,
  c.full_name,
  a.account_no,
  a.status as account_status_code,

  s.available_balance,
```

### 题4：按账户统计“总入账 / 总出账 / 净额”

**业务意义**：账户行为分析、对账预检查。

**关键点**：

- `direction = 1/-1`
- `FILTER`（PostgreSQL 很好用）
- 只统计成功流水 `status=1`

```sql
select
  a.account_no,
  count(*) filter (where t.status = 1) as posted_txn_count,
  coalesce(sum(t.amount) filter (where t.status = 1 and t.direction = 1), 0) as total_in,
  coalesce(sum(t.amount) filter (where t.status = 1 and t.direction = -1), 0) as total_out,
  coalesce(sum(t.amount * t.direction) filter (where t.status = 1), 0) as net_amount
from yewu.account a
left join yewu.txn t
  on t.account_id = a.account_id
group by a.account_no
order by a.account_no;
```

### 题7：检查转账主单是否存在“借贷成对分录”

**业务意义**：账务完整性校验（非常真实）。

**思路**：

- 每个 `transfer_order_id`
- 统计 `D`、`C` 条数
- 看金额是否平衡（成功单一般要平）

```sql
select
  o.order_no,
  o.status as order_status,
  count(e.entry_id) as entry_cnt,
  count(*) filter (where e.dr_cr = 'D') as d_cnt,
  count(*) filter (where e.dr_cr = 'C') as c_cnt,
  coalesce(sum(case when e.dr_cr = 'D' then e.amount else 0 end), 0) as d_amt,
  coalesce(sum(case when e.dr_cr = 'C' then e.amount else 0 end), 0) as c_amt
from yewu.transfer_order o
left join yewu.ledger_entry e
  on e.transfer_order_id = o.transfer_order_id
group by o.order_no, o.status
order by o.order_no;
```

> 你会看到：
> 
> - 成功单：通常 D/C 成对
> - 处理中：可能 pending 分录
> - 失败单：可能是 reversed 示例（不一定成对）

这就是现实系统里“状态驱动账务”的雏形。

### 题11：按交易类型统计成功流水金额和笔数

**业务意义**：日报/分析报表基础。

